# Scene Manager

> **Status**: In Design
> **Author**: user + game-designer
> **Last Updated**: 2026-04-05
> **Implements Pillar**: "20분의 밀도" (Dense Runs)

## Overview

Scene Manager는 게임의 모든 씬 전환을 관리하는 Foundation 시스템이다. 마을↔던전 진입/퇴장, 던전 내 방(room) 간 이동, 로비/메뉴 전환을 담당한다. 씬 로딩/언로딩, 전환 이펙트(페이드 등), 로딩 중 게임 상태 보존을 책임진다. 던전 런의 "밀도"를 유지하기 위해 방 전환은 최대한 빠르고 심리스해야 한다.

## Player Fantasy

"흐름이 끊기지 않는다." 던전 방을 클리어하고 다음 방에 진입하는 순간, 로딩 화면 없이 매끄럽게 이어진다. 마을에서 던전으로의 전환도 짧은 연출과 함께 자연스럽게. 20분 런 동안 "기다리는 시간"은 제로 — 모든 전환이 게임플레이의 일부처럼 느껴진다.

## Detailed Design

### Core Rules

1. **씬 타입 3종**:
   - `Town` — 마을/로비. 영구 씬, 던전 밖 활동의 허브
   - `DungeonRoom` — 던전 방. Dungeon Generator가 생성한 방 데이터를 인스턴스화
   - `UI` — 전체 화면 UI (인벤토리, 설정 등). 게임 씬 위에 오버레이

2. **던전 방 전환 (심리스)**:
   - 플레이어가 출구 문에 접촉 → 짧은 페이드(0.2~0.3초) → 다음 방 로드 → 페이드 인
   - 현재 방 언로드 + 다음 방 로드를 페이드 중에 처리
   - 방 프리로딩: 현재 방 진입 시 다음 방 후보를 백그라운드 로드(`ResourceLoader.load_threaded_request`) 시작

3. **마을↔던전 전환**:
   - 던전 진입: 페이드 아웃(0.5초) → 로딩 화면(팁 텍스트 표시) → 첫 방 로드 → 페이드 인
   - 던전 퇴장(클리어/사망/포기): 결과 화면 → 페이드 → 마을 복귀
   - 최대 로딩 시간 목표: 2초 이내 (모바일 기준)

4. **씬 스택**: 현재 활성 게임 씬 + UI 오버레이를 스택으로 관리. UI 씬은 게임 씬 위에 push/pop 방식

5. **상태 보존**: 씬 전환 시 플레이어 상태(HP, 인벤토리, 버프, 쿨다운)를 보존하여 새 씬에 전달. 씬 간 공유 데이터는 전역 GameState 오토로드에 저장

6. **전환 이펙트**: 페이드(기본). 커스텀 이펙트(서클 와이프 등)는 폴리시 단계에서 확장 가능

### States and Transitions

| State | 설명 | 전환 조건 |
|-------|------|-----------|
| **Idle** | 현재 씬 활성, 전환 없음 | 기본 상태 |
| **FadingOut** | 화면 어두워지는 중 | 전환 요청(change_scene) 수신 시 |
| **Loading** | 다음 씬 로드 중 (로딩 화면 표시) | 페이드 아웃 완료 시 |
| **FadingIn** | 새 씬에서 화면 밝아지는 중 | 로드 완료 시 |
| **Preloading** | 백그라운드에서 다음 방 프리로드 중 | 방 진입 시 자동 시작 |

- **기본 흐름**: Idle → FadingOut → Loading → FadingIn → Idle
- **프리로드 완료 시**: Loading 단계 스킵 가능 (FadingOut → FadingIn 직행)
- **전환 중**: Input System에 `disable_input()` 요청 → 플레이어 입력 차단
- **전환 중 추가 전환 요청**: 무시 (현재 전환 완료까지 큐잉하지 않음)

### Interactions with Other Systems

| System | 방향 | 인터페이스 |
|--------|------|-----------|
| **Dungeon Generator** | ← 입력 | 다음 방 데이터(씬 경로, 몬스터 스폰 데이터, 출구 위치) 수신 |
| **Camera System** | → 출력 | 방 전환 시그널(`room_changed`) 발송 → 카메라 Transitioning 진입 |
| **Input System** | → 출력 | 전환 시작 시 `disable_input()`, 전환 완료 시 `enable_input()` |
| **Player Controller** | ↔ 양방향 | 플레이어 상태 보존/복원, 새 방 스폰 포인트에 배치 |
| **HUD** | → 출력 | 로딩 화면/팁 표시 요청, 전환 완료 시그널 |
| **Save/Load** | ↔ 양방향 | 던전 퇴장 시 세이브 트리거, 로드 시 씬 복원 요청 수신 |
| **Town/Lobby** | ← 입력 | 던전 진입/마을 복귀 요청 수신 |

## Formulas

### 1. 페이드 타이밍

| 전환 타입 | Fade Out | Loading | Fade In | 총 시간 (프리로드 성공) |
|-----------|----------|---------|---------|-------------------------|
| 방 전환 | 0.2초 | 0초 (프리로드) | 0.2초 | 0.4초 |
| 마을→던전 | 0.5초 | 가변 | 0.5초 | 1.0초 + 로딩 |
| 던전→마을 | 0.5초 | 가변 | 0.5초 | 1.0초 + 로딩 |

### 2. 프리로드 타이밍

```
preload_start_time = room_enter_time + PRELOAD_DELAY
# PRELOAD_DELAY: 0.5초 (방 진입 직후 프레임 스파이크 방지)
```

### 3. 로딩 진행률

```
progress = ResourceLoader.load_threaded_get_status(path, progress_array)
# progress_array[0]: 0.0 ~ 1.0
```

### 4. 페이드 알파

```
# Fade Out
alpha = elapsed / fade_duration  # 0→1 (투명→불투명)
# Fade In
alpha = 1.0 - (elapsed / fade_duration)  # 1→0 (불투명→투명)
```

## Edge Cases

1. **프리로드 실패**: 다음 방 프리로드 실패 시 동기 로드로 폴백하며 로딩 화면 표시
2. **전환 중 플레이어 사망**: 사망 처리 우선, 진행 중인 전환 취소. 사망 연출 후 결과 화면으로 전환
3. **빠른 방 연속 통과**: 이전 전환 완료 전 새 전환 요청은 무시. 한 번에 한 전환만 처리
4. **메모리 부족 (모바일)**: 이전 방 언로드 → 다음 방 로드 순서 보장. 동시 2방 이상 메모리 점유 금지
5. **앱 백그라운드 전환**: 전환 중 앱이 백그라운드로 가면 전환 일시정지, 포그라운드 복귀 시 재개
6. **마을 씬 메모리**: 던전 진입 시 마을 씬 언로드 (메모리 절약). 복귀 시 재로드
7. **UI 스택 오버플로**: 최대 UI 오버레이 `MAX_UI_STACK`(3)개로 제한. 초과 요청 시 경고 로그

## Dependencies

### Upstream (이 시스템이 의존하는 시스템)

없음 — Foundation 레이어. 독립 동작.

### Downstream (이 시스템에 의존하는 시스템)

| System | 의존 유형 | 설명 |
|--------|-----------|------|
| **Dungeon Generator** | Hard | 방 데이터 생성 후 Scene Manager에 로드 요청 |
| **Town/Lobby System** | Hard | 마을↔던전 전환 요청 |
| **Player Controller** | Soft | 전환 시 플레이어 스폰 포인트 배치 |
| **Camera System** | Soft | 방 전환 시그널 수신하여 카메라 전환 |
| **Input System** | Soft | 전환 중 입력 차단/해제 요청 수신 |

## Tuning Knobs

| Knob | 기본값 | 안전 범위 | 영향하는 게임플레이 | 극단값 효과 |
|------|--------|-----------|---------------------|-------------|
| `room_fade_duration` | 0.2초 | 0.1 ~ 0.5초 | 방 전환 체감 속도 | 너무 짧으면 번쩍임, 너무 길면 흐름 끊김 |
| `town_fade_duration` | 0.5초 | 0.3 ~ 1.0초 | 마을↔던전 전환 연출 | 너무 짧으면 급작스러움, 너무 길면 지루 |
| `preload_delay` | 0.5초 | 0.0 ~ 1.0초 | 프리로드 시작 타이밍 | 0이면 진입 스파이크, 높으면 프리로드 미완료 |
| `max_loading_time` | 2.0초 | 1.0 ~ 5.0초 | 이 시간 초과 시 스피너 표시 | 너무 짧으면 불필요한 스피너, 너무 길면 대기 |
| `max_ui_stack` | 3 | 2 ~ 5 | 동시 UI 오버레이 제한 | 너무 적으면 UI 전환 제약, 너무 많으면 메모리 |

## Visual/Audio Requirements

- 페이드: 검정 화면으로 페이드 인/아웃 (CanvasLayer 최상위)
- 로딩 화면: 게임 로고 또는 지역 일러스트 + 팁 텍스트 + 간단한 로딩 인디케이터
- 방 전환 시 짧은 문 열림 SFX (Combat Feedback 또는 Audio Manager 담당)
- 마을↔던전 전환 시 환경음 크로스페이드 (Audio Manager 담당)

## UI Requirements

- 로딩 화면에 랜덤 팁 텍스트 표시 (게임 메카닉, 로어, 조작법)
- 로딩 진행률 표시: 프로그레스 바 또는 회전 아이콘
- 페이드 레이어는 모든 UI 위에 렌더링 (z-index 최상위)

## Acceptance Criteria

1. 던전 방 전환이 프리로드 성공 시 0.4초 이내 완료 (페이드 포함)
2. 마을→던전 전환이 2초 이내 완료 (모바일 중간 사양 기준)
3. 전환 중 플레이어 입력이 완전 차단되고, 전환 완료 후 복귀
4. 전환 전후 플레이어 상태(HP, 버프, 인벤토리, 쿨다운) 온전히 보존
5. 이전 방 메모리가 다음 방 로드 전에 해제 (동시 2방 점유 없음)
6. 프리로드 실패 시 동기 로드로 정상 폴백, 크래시 없음
7. 전환 중 추가 전환 요청이 무시되고 에러 없음
8. UI 오버레이 push/pop이 정상 동작 (스택 순서 유지)
9. Scene Manager 자체의 프레임당 오버헤드 0.2ms 이내 (전환 중 제외)
